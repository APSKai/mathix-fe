# SPC Frontend

## Requirements

- Node.js >= 20 (LTS) and npm or Yarn.
- Git
- Network access to the backend API if the project needs to connect to a server (configured via `.env`).

Recommended versions (verified with this repo):

- Node 20.x
- npm 9.x or Yarn 1/2+

## Environment

- Copy `.env.example` to `.env` or `.env.development` (using with development mode)

```
cp .env.example .env
```

- Place environment variables in a `.env` file at the project root.
- Example `.env`:

```dotenv
VITE_APP_ROOT_API=http://localhost:8000/api
```

Update `VITE_APP_ROOT_API` to point to your backend URL when running in non-local environments.

## Setup

Open PowerShell (Windows) in the project folder `e:\code\spc-frontend` and run:

If you use npm:

```powershell
npm install
```

If you use Yarn:

```powershell
yarn install
```

## Development (Run locally)

Start the development server with hot reload:

```powershell
npm run dev
# or
yarn dev
```

By default Vite serves the app at `http://localhost:5173` (port may vary). Open that address in your browser.

## Build & Preview (Production locally)

Build the production bundle:

```powershell
npm run build
# or
yarn build
```

Preview the built site locally:

```powershell
npm run serve
# or
yarn serve
```

## Lint & Format

Run ESLint to check the code:

```powershell
npm run lint
# or
yarn lint
```

Format code with Prettier and apply ESLint fixes:

```powershell
npm run format
# or
yarn format
```

## Important NPM scripts (summary)

- `dev` — start Vite in development mode.
- `build` — build the app for deployment.
- `serve` — preview the production build locally.
- `lint` — run ESLint checks.
- `format` — run Prettier and ESLint fixes.

## Deployment notes

- Ensure `VITE_APP_ROOT_API` is configured correctly in your environment or CI/CD so the app points to the right backend.
- For production hosting, serve the `dist` folder generated by `vite build` with a static file server or configure a reverse proxy (nginx) to serve the files.
